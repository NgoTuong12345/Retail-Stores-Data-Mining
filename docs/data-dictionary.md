### `dim_company`

VN MST-grain company dimension, unifying curated owners (brand_master.yaml owner_msts) with observed operators (stores.base_mst) ‚Äî the two 'legal entity behind a brand' representations.

| Column | Type | Description |
|---|---|---|
| `mst` | VARCHAR | 10-digit VN tax code (company legal-entity key). |
| `company_name` | VARCHAR | Curated owner name if present, else a sampled DKKD registration name for this MST. |
| `is_curated_owner` | BOOLEAN | True if this MST appears in brands/_master/brand_master.yaml's owner_msts. |
| `is_observed` | BOOLEAN | True if this MST appears as an observed operator (stores.base_mst) in the DKKD scrape. |

### `bridge_brand_entity`

One row per (brand_slug, mst, role): role is 'owner_curated' | 'operator_observed' (or both, resolved to 'owner_operator' in v_brand_entities). n_locations is the observed DKKD license count for that (brand, mst) pair.

| Column | Type | Description |
|---|---|---|
| `brand_slug` | VARCHAR | Brand directory slug. |
| `mst` | VARCHAR | 10-digit VN tax code ‚Äî FK to dim_company.mst. |
| `role` | VARCHAR | 'owner_curated' (from brand_master.yaml) | 'operator_observed' (from stores.base_mst). |
| `n_locations` | INTEGER | Observed DKKD license count for this (brand, mst) pair; only meaningful for operator_observed rows. |
