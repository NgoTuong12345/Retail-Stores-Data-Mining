## 1. Overview

After 19 minutes of inactivity, a modal popup appears on screen with a 60-second countdown warning the user the session is about to end. If the user does not click "Tiếp tục phiên làm việc" (Continue Session), the session is terminated at T = 20 minutes.

The popup is generated by a custom jQuery plugin `$.fn.timeout` that is bundled inside `/inf/JavaScript/Assets.ashx`. Inspecting the DOM at runtime reveals the full structure and selectors below.

(function keepSessionAlive() {

### 6.2 Selenium (Python)

from selenium.webdriver.common.by import By
import time, threading

def keep_session_alive(driver):
    while True:
        try:
            modal = driver.find_element(By.ID, "tm-modal")
            if modal.is_displayed():
                keep_btn = driver.find_element(By.ID, "tm-btn-keep")
                if keep_btn.is_displayed():

            pass

## 7. How It Works Under the Hood

1. The plugin reads the `fw_auth` cookie to check the session timestamp.
2. A background timer compares the cookie value to `tmTimeout`.
3. When remaining time ≤ `tmWarningTime` (60s), it creates and injects `#tm-modal` into `document.body`.
4. A second interval updates the countdown text every second.
5. Clicking `#tm-btn-keep` triggers an AJAX POST to `/inf/Public/Srv.aspx/KeepSession`, which refreshes the server session and resets the `fw_auth` cookie.
6. Clicking `#tm-btn-logout` or waiting for the countdown to reach 0 transitions the modal into the "Expired" state, requiring the user to re-authenticate.
